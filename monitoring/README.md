# Monitoring

Ready-to-use Prometheus + Grafana setup for the NIXI SBC.

The SBC exposes Prometheus metrics at `GET /metrics` on its management API
(behind the bearer token). All series are prefixed `sbc_` — calls, SIP
traffic (by method / by response code), auth, anti-fraud (bans, IRSF
destination blocks, per-user limits), media (RTP/SRTP/transcoding) and
health (last-CDR age, RTP timeouts).

## Prometheus

```yaml
# prometheus/prometheus-sbc.yml — add this scrape job to your prometheus.yml
scrape_configs:
  - job_name: 'nixi-sbc'
    metrics_path: /metrics
    authorization:
      type: Bearer
      credentials: 'YOUR_API_TOKEN'
    static_configs:
      - targets: ['127.0.0.1:8080']
```

Optional alerting rules are in `prometheus/alert_rules.yml`
(`SBCDown`, `SBCCdrsStale`, `SBCRtpTimeouts`, `SBCAuthFailureSpike`).
Validate with `promtool check rules prometheus/alert_rules.yml`.

## Grafana

1. Copy the datasource provider to Grafana:
   `grafana/provisioning-datasource.yaml → /etc/grafana/provisioning/datasources/`
   (or use your existing Prometheus datasource — then set its UID in the
   dashboard, see below).
2. Copy the dashboard provider:
   `grafana/provisioning-dashboards.yaml → /etc/grafana/provisioning/dashboards/`
3. Copy the dashboard JSON to the provider's `path`:
   `grafana/dashboards/sbc-overview.json → /var/lib/grafana/dashboards/`
4. `systemctl restart grafana-server` — the dashboard appears in the **SBC**
   folder as **NIXI SBC — Overview**.

### Datasource UID

The dashboard references the Prometheus datasource by UID
(`efdqq22ymisqod` — the value used by the provided datasource file). If you
point the dashboard at an existing datasource with a different UID, either
set that UID in `provisioning-datasource.yaml`, or replace the `uid` fields
in `dashboards/sbc-overview.json`.

### Regenerating the dashboard

The dashboard JSON is generated by `grafana/dashboards/generate.py` (pure
Python, no deps). Edit the panel definitions there and run:

```bash
python3 grafana/dashboards/generate.py > grafana/dashboards/sbc-overview.json
```
